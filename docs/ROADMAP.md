
**关键文件位置**:
- 改 UI: `templates/analysis.html` + `app.py` 新增 `/api/analyze/<code>/verdict`
- 改算法: `scripts/analyze.py` 新增 `detect_adjustment_vs_reversal()`
- 改数据: `database/models.py` + `database/db_manager.py`

### 目标
解决用户三大痛点:
1. 买点不好 → 回调拿不住
2. 短期调整 → 卖飞股票
3. 不会高抛低吸控制仓位

### 任务清单

#### 2.1 综合判定卡片 (顶部)
**预计工作量**: 半天

- [ ] 创建 `/api/analyze/<code>/verdict` 端点
- [ ] 综合判定逻辑:
  - 锁仓评分 ≥ 5/6 → 倾向"持有"
  - 派发评分 ≥ 4/5 → 倾向"卖出风险"
  - 背离强度 + 形态 → 综合信心度
- [ ] 在 `analysis.html` 顶部添加大卡片
- [ ] 视觉: 大字号颜色编码 (绿/黄/红)

#### 2.2 三维评分卡
**预计工作量**: 1 天

- [ ] 结构健康度评分 (TPC + 形态 + 偏度)
- [ ] 主力动向评分 (锁仓 + 行为 + 背离)
- [ ] 风险评估 (派发 + 顶部信号)
- [ ] 在判定卡片下方 3 列布局

#### 2.3 持仓评估模式
**预计工作量**: 1 天

- [ ] 检测当前持仓股 (用 `/api/positions/by_code/<code>`)
- [ ] 持仓模式 UI: 显示入场价 + 当前价 + 浮盈
- [ ] 加仓/减仓建议: 基于支撑阻力 + 筹码变化

#### 2.4 算法信号回放
**预计工作量**: 2 天

- [ ] 在每笔 trade_log 的入场日跑一次 `analyze`
- [ ] 存 `algorithm_signal` (JSON 快照)
- [ ] 异步任务队列 (避免阻塞, 已有 rate limit)
- [ ] 批量回放脚本: `scripts/backfill_algorithm_signals.py`

#### 2.5 调整 vs 反转识别
**预计工作量**: 1 天

- [ ] 新增算法函数: `detect_adjustment_vs_reversal(metrics)`
- [ ] 逻辑:
  - 调整 = 缩量 + 筹码仍集中 + 锁仓未破
  - 反转 = 放量下跌 + 筹码分散 + 派发
- [ ] 在持仓评估模式中突出显示

### 验收标准
- 打开任意持仓股, **3 秒内看到该买/持/卖建议 + 信心度**
- 调整时显示"建议持有, 主力未动"而不是单纯的"跌了"
- 反转时显示"建议清仓"而不是模糊的"高位风险"

## Phase 3: 适配度回测

### 目标
量化"算法对你的选股模式的适配度"

### 任务清单

#### 3.1 算法 vs 实际对照表
**预计工作量**: 1 天

- [ ] 用 trade_logs 跑算法 (在每笔 entry_date)
- [ ] 对比:
  - 算法说"买入信号" vs 你实际买了
  - 算法说"持有" vs 你实际卖了 → 标记"潜在卖飞"
  - 算法说"卖出" vs 你还持有 → 标记"潜在风险"
- [ ] 报告: 每个错过的潜在机会

#### 3.2 关键指标计算
**预计工作量**: 1 天

- [ ] 卖飞次数: 算法说持有, 你卖了, 后来 5 日内涨 ≥ 5%
- [ ] 回调割肉次数: 算法说锁仓良好, 你卖了, 后来 5 日内反弹 ≥ 5%
- [ ] 入场点改善: 算法建议入场价 vs 实际入场价

#### 3.3 报告生成
**预计工作量**: 半天

- [ ] `/api/backtest/validation/<account>` 端点
- [ ] 报告页面: `templates/validation.html`
- [ ] 时间维度: 按月统计 (看市场风格变化)

### 验收标准
- 一份"算法 vs 你"的差距报告
- 数字告诉你算法能减少多少"卖飞"和"回调割肉"

## Phase 4: 进化引擎

### 目标
让算法持续适配你的选股模式 + 市场风格变化

### 任务清单

#### 4.1 基于实际结果的参数优化
**预计工作量**: 2 天

- [ ] 用 Phase 3 的错例作为 fitness function
- [ ] 进化引擎搜索参数空间:
  - 锁仓评分阈值 (4/5/6)
  - 背离权重 (40%/30%/20%/15%)
  - ATR 阈值
- [ ] 输出: 你的选股模式最优参数

#### 4.2 市场状态识别
**预计工作量**: 3 天

- [ ] 跟踪近期 20 日大盘指标
  - 涨停数量 / 跌停数量
  - 成交量均值
  - 北向资金净流入
- [ ] 状态分类: 趋势市 / 震荡市 / 题材市
- [ ] 不同状态用不同参数组合

#### 4.3 漂移检测
**预计工作量**: 1 天

- [ ] 跟踪算法表现 vs baseline
- [ ] 表现下滑阈值: -15% 胜率 / -2% 平均收益 → 触发警告
- [ ] UI: 在主页显示"算法表现可能下降, 建议重新优化"

#### 4.4 自动重训练
**预计工作量**: 2 天

- [ ] 检测到漂移后, 自动触发参数搜索
- [ ] 跑完测试 → 如果新参数在 holdout 上表现更好 → 应用
- [ ] 通知用户

### 验收标准
- 算法的"参数自适应"在 3 个月内表现稳定
- 市场风格切换时, 自动检测 + 重新优化 ≤ 7 天

## 🔑 关键技术决策

1. **不删除 watchlist.active 字段**: 软删除 → 硬删除已切换, 但保留字段避免 schema 迁移
2. **algorithms_signal 不实时计算**: 每次访问时跑一次 analyze (有 rate limit + 缓存)
3. **历史数据不动**: trade_logs 一旦写入就锁定, 算法升级不影响历史
