## 🏗️ 当前架构

```
数据层 (Data Layer)
  ├ Tushare API
  ├ 分段获取算法
  └ 数据缓存

指标层 (Indicator Layer)
  ├ 基础指标 (TPC, Width, TP3...)
  ├ 高级指标 (Skewness, Kurtosis, Entropy...)
  └ 衍生指标 (变化率, 相对位置...)

信号层 (Signal Layer)
  ├ 单指标信号 (25+)
  ├ 组合信号 (6+)
  └ 信号规则库

决策层 (Decision Layer)
  ├ 建仓决策
  ├ 持仓管理 (V3)
  └ 风险控制

展示层 (Presentation Layer)
  ├ Flask Web
  ├ 仪表盘
  └ 复盘中心
```

## 🎓 核心方法论

### 1. 定性优先
- 每个信号必须有定性理论
- 避免纯数据挖掘
- 跨股验证稳定性

### 2. 分级管理
- 三级减仓机制
- 多重过滤
- 风险可控

### 3. 反直觉洞察
- 出货时 Winner 接近 100%
- 见顶时 TP3/Kurtosis 最高
- 上涨中 Winner 下降 = 洗盘

### 4. 行情适配
- 单边上涨: 持有
- 单边下跌: 信号减仓
- 震荡市: 信号择时

## 📊 验证标准

任何新功能上线前必须通过：

| 测试 | 标准 |
|------|------|
| 单元测试 | 覆盖率 > 80% |
| 跨股验证 | > 5 只股票 |
| 不同时间窗口 | 至少 2 个时间窗口 |
| 定性理论 | 明确的理论支撑 |
| 实盘模拟 | 至少 1 个月模拟盘 |

## ⚠️ 重要提醒

### 1. 实盘谨慎
- 当前是**研究和回测工具**
- 实盘前需做至少 1 个月模拟盘
- 注意滑点、流动性、税费

### 2. 参数个性化
- 不同股票特性不同
- 不要用一套参数打天下
- 建议先观察再使用

### 3. 数据质量
- Tushare 数据有延迟
- 筹码数据从 20260420 开始
- 历史数据需额外处理

### 4. 风险控制
- 任何策略都有失效的时候
- 设置硬止损
- 控制单只股票仓位
