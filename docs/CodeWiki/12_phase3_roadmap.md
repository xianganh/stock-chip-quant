#### 4.2 面板 A: 算法 vs 实际走势图

**目标**：在选定股票的时间段上，叠加显示
- 实际价格曲线
- 算法每日 verdict（持有/减仓/清仓/观望）背景色
- 算法关键指标变化（背离触发、锁仓评分突变）
- 用户的实际交易（买卖点）

**实现**：
- 前端：Plotly 时间序列 + 背景色块 + 标记点
- 数据：`GET /api/replay/timeline/<ts_code>?start=...&end=...`

#### 4.3 面板 B: 胜率与偏差分析

**目标**：统计每个算法指标的胜率分布

**核心问题**：
- "lock_score >= 5/6 时胜率多少？"
- "P1_pct 在什么区间胜率最高？"
- "4 类背离哪种在历史数据上最有效？"

**实现**：
- 分组统计（按指标分桶）
- 柱状图 / 散点图
- 自动给出"高胜率区间"建议

#### 4.4 面板 C: 决策时刻信号快照

**目标**：列表展示每笔历史交易当时的算法判断

| 股票 | 入场日 | 算法 verdict | 信心度 | 锁仓 | 派发 | 实际盈亏 | 算法对错 |
|------|--------|--------------|--------|------|------|---------|---------|
| 603039 | 20240401 | 持有 | 80% | 5/6 | 0 | +12% | ✅ |
| 002371 | 20240515 | 观望 | 50% | 2/6 | 2 | -5% | ⚠️ 跳过 |
| ... | ... | ... | ... | ... | ... | ... | ... |

**交互**：
- 点击某行 → 跳转到面板 A 的时间线视图
- 筛选：算法对 / 算法错 / 跳过
- 排序：按盈亏、按时间、按指标

#### 4.5 面板 D: 参数敏感性热力图

**目标**：展示参数变化对结果的影响

**示例**：
```
            cmf_threshold=0.03  0.05  0.08  0.10  0.15
tpc=65     [胜率%]              ...
tpc=70                          ...
tpc=75                          ...
tpc=80                          ...
tpc=85                          ...
```

**实现**：
- 回测引擎扫描 5×5=25 组参数
- Plotly heatmap
- 高亮显示当前推荐参数

## 🚦 关键决策点

### 决策 1: 回放粒度
- **方案 A**: 按 trade_log 的入场日单次回放
- **方案 B**: 按 position 的整个生命周期（入场→出场）逐日回放
- **推荐**: 方案 B，更接近实战

### 决策 2: 异步任务
- **方案 A**: 用 Celery（重型，需要 Redis/RabbitMQ）
- **方案 B**: 用 Python threading + 进度查询（轻量）
- **推荐**: Phase 3 用方案 B，未来如需扩展再换 Celery

### 决策 3: 敏感性扫描范围
- **方案 A**: 单参数扫描（5×5=25 组）
- **方案 B**: 多参数扫描（5×5×5×5=625 组）
- **推荐**: 方案 A 起步，按需扩展
