## 🎯 项目一句话定义

**短线投资者的持仓决策辅助工具** — 分析筹码峰 + 技术面 + 资金面，回答"该不该买/该不该卖"。

## 🧑‍💼 用户画像

- **短线为主**，持仓 1 周以内 (大牛股除外)
- 算法对**所有股票统一适用**，不按"涨价/卡脖子"等类型分类
- **14 日分析窗口**用于验证涨停后行情，不代表持仓期
- **Layer 0 选股在外部做**（本项目只看已经感兴趣的股票）
- **三大痛点**:
  - 买点不好回调拿不住
  - 短期调整卖飞
  - 不会高抛低吸

## 🏗️ 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 0: 选股（用户的职责，项目外）                           │
│   从 5000 只股票中找到「有预期差」的票                        │
└─────────────────────────────────────────────────────────────┘
                            ↓ 选股清单
┌─────────────────────────────────────────────────────────────┐
│ Layer 2.5: 本项目职责                                        │
│   给定一只股票，回答 6 个问题：                                │
│   1. 结构是否健康？     2. 主力是否还在？                      │
│   3. 有没有出货嫌疑？   4. 买卖点建议？                        │
│   5. 关键支撑阻力位？   6. 该继续持有还是该卖？                 │
└─────────────────────────────────────────────────────────────┘
                            ↓
              Flask Web UI / REST API / CLI
```

## 🔧 技术栈

- **Web 框架**: Flask 3.0 + Flask-SQLAlchemy 3.1
- **数据源**: Tushare Pro（cyq_chips / stk_factor / moneyflow / daily_basic）
- **数据处理**: pandas / numpy
- **可视化**: Plotly（前端 JS）+ Kaleido（静态导出）
- **AI 解读**: CodeBuddy 配置的 LLM（OpenAI 兼容协议）
- **数据库**: SQLite（本地）
- **测试**: pytest
