# 01. 项目概述

## 📌 一句话定义

> 短线投资者的持仓决策辅助工具 — 分析筹码峰 + 技术面 + 资金面，回答"该不该买/该不该卖"。

## 🎯 项目定位

**这不是一个通用选股工具，而是持仓决策辅助系统。**

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 0: 选股（用户的职责，项目外）                           │
│   从 5000 只股票中找到「有预期差」的票                       │
│   触发方式: 涨停/产业链/信息挖掘/...                         │
└─────────────────────────────────────────────────────────────┘
                            ↓ 选股清单
┌─────────────────────────────────────────────────────────────┐
│ Layer 2.5: 本项目职责                                       │
│   给定一只股票，回答 6 个问题：                                │
│   1. 结构是否健康？                                          │
│   2. 主力是否还在？（吸筹/洗盘/派发？）                       │
│   3. 有没有出货嫌疑？                                        │
│   4. 买卖点建议？                                            │
│   5. 关键支撑阻力位？                                        │
│   6. 该继续持有还是该卖？                                     │
└─────────────────────────────────────────────────────────────┘
```

## 🧑‍💼 用户画像

| 维度 | 描述 |
|------|------|
| 持仓周期 | 短线为主，1 周以内（大牛股除外） |
| 算法偏好 | 对所有股票统一适用，**不按"涨价/卡脖子"等类型分类** |
| 分析窗口 | 14 日（用于验证涨停后行情，**不代表持仓期**） |
| 选股方式 | **Layer 0 选股在外部做**（本项目只看已经感兴趣的股票） |
| 三大痛点 | 1. 买点不好回调拿不住  2. 短期调整卖飞  3. 不会高抛低吸 |

## ✨ 核心功能

### 1. 选股池管理
- 按预期差类型分类（涨价/订单/卡脖子/周期反转/估值调整）
- 双向反查：输入代码自动填名称，输入名称自动填代码
- 一键硬删除（替代之前的"移除 → 永久删除"两步流程）
- 数据格式统一：自动 normalize 加 `.SZ`/`.SH` 后缀

### 2. 筹码峰演化分析
- P1/P2/P3 峰位追踪
- TPC（Top3 Concentration）三峰集中度
- 偏度 / 峰度 / 熵（分布形态量化）
- 局部极大值峰值检测 + 形态分类
- 阻力位 / 支撑位识别

### 3. 价格-筹码背离检测
四类背离（v2.3 起）：
- **价格-筹码背离**（暗中建仓）— 权重 35%
- **资金-价格背离**（压盘吸筹）— 权重 30%
- **形态-认知背离**（表散实聚）— 权重 20%
- **时间-认知背离**（筹码领先价格）— 权重 15% 🆕

### 4. 经典量化指标
- CMF（蔡金资金流）— 替代裸 net_mf_amount
- ADX（平均趋向指数）— 趋势强度 + 方向
- ATR（平均真实波幅）— 波动率分位

### 5. 锁仓判定 + 派发评分
- **6 条件锁仓判定**（P1 稳定 + tp3 提升 + dist 转正 + Top5 趋势 + winner 趋势 + TPC 趋势）
- **5 维派发评分**（D1 P1 反向 + D2 成本反向 + D3 Top5 分散 + D4 价格破成本 + D5 内部人）

### 6. 回测引擎
- 事件驱动时序回测
- 支持 4 类信号（locking / divergence / dispatch / build_divergence）
- 止损/止盈/时间出场三种退出逻辑
- 计算胜率、夏普、最大回撤等指标

### 7. 进化引擎
- 参数网格搜索自动优化
- 跨参数组合寻找最优配置
- 与大盘基线（buy & hold）对比

### 8. AI 解读（LLM）
- 调用 CodeBuddy 配置的 LLM
- 生成 4 部分 markdown 解读：核心结论 / 筹码面 / 技术面 / 操作建议
- 缓存避免重复调用，60s 限流避免狂点浪费 token

### 9. 持仓 + 交易日志
- 导入券商交易文件（FIFO 推导持仓）
- 算法信号快照（每笔交易关联买入时的算法判断）
- 历史胜率与盈亏统计

### 10. 知识图谱集成
- 扫描项目内 `knowledgeGraph/` 目录
- 解析 YAML frontmatter 建立索引
- 支持按代码 / 按 tag 关联查询

## 🚫 不做的事（明确边界）

| 不做 | 原因 |
|------|------|
| 通用选股 | 用户已在外部 Layer 0 完成 |
| 按预期差类型分类算法 | 用户明确说分类不准确，反而误导 |
| 长期投资 | 用户为短线风格 |
| 自动下单 | 决策辅助工具，非交易系统 |
| 实时盯盘 | 14 日分析窗口验证，非实时交易 |

## 📝 关键设计决策

> 这些决策经过用户长期使用验证，**新功能不要违背**

1. **不按"预期差类型"分类算法** — 用户明确说分类不准确，反而误导
2. **14 日分析窗口 ≠ 持仓期** — 前者用于验证，后者是用户实际交易周期
3. **数据格式统一 `.SZ`/`.SH` 后缀** — 自动 normalize，不接受 6 位数字裸码
4. **删除是硬删除** — 不再有"移除 → 恢复"软删除，trade_logs 保留
5. **Layer 0 选股在外部** — 本项目只看已经感兴趣的股票
6. **算法信号存在 trade_log.algorithm_signal** — 每笔交易关联当时的算法判断
